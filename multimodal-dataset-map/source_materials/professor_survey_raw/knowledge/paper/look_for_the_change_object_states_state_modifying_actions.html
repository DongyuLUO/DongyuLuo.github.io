Look for the Change: Learning Object States and State-Modifying Actions from Untrimmed Web Videos

Tomáš Souček, Jean-Baptiste Alayrac, Antoine Miech, Ivan Laptev, Josef Sivic · CIIRC CTU / DeepMind / ENS-Inria · CVPR 2022 · arXiv:2203.11637 · PDF

One-liner. A self-supervised model that, given a noisy collection of long untrimmed web videos of one state-modifying action (e.g. "cut apple"), temporally localizes the initial object state → manipulating action → end object state triple in each video — supervised only by the causal arrow of time plus a handful of exemplar still images — and introduces ChangeIt, a 2,600+-hour / 34k-state-change dataset to scale this past the small curated setups of prior work.

Problem & motivation

Human actions induce changes in object state ("cut an apple", "pour coffee", "fill a cup"), and recognizing both the action and the resulting state change is a prerequisite for following recipes or reproducing actions in the world. Prior work on object-state / state-modifying-action recognition was confined to small-scale curated setups: Alayrac et al. [2] used closed-form linear classifiers over only seven object-action classes with tens of videos each; Damon et al.'s task-fluent / "i-learn" line [18] worked on a single constrained egocentric scene. The open question this paper poses: can learning of object states and their state-modifying actions be scaled up to noisy, uncurated web video with minimal supervision? The difficulty is twofold — the huge variability of object-action pairs in the wild, and the cost of manually collecting and annotating temporal boundaries.

Method

Three contributions stack on each other (Fig 2):

1. Causal-ordering self-supervision. The model learns two per-frame classifiers over second-by-second visual features: an action classifier g (confidence the frame depicts the manipulating action) and a state classifier h outputting two scores h1, h2 (initial vs. end state). No ground-truth temporal labels exist; instead the free supervisory signal is the arrow of time — the action must appear between the initial and end states. Training alternates (i) optimizing g, h given current per-video labels l(v), and (ii) re-estimating l(v) as the action/state temporal positions that maximize the classifier scores subject to the causal ordering constraint 1 ≤ s1 < a < s2 ≤ Tv (initial-state index before action index before end-state index). Because most state-modifying actions are physically irreversible (cut apple) or conventionally one-directional (clean shoes), this ordering is a strong, free inductive bias. The "no-action" negatives are sampled as hard negatives at a fixed temporal distance κ (~60s) from the positives rather than uniformly — a choice the ablations show is crucial.

2. Noise-adaptive weighting from a few exemplar images. Uncurated web video for a query like "cutting an apple" is full of irrelevant clips (e.g. videos of Apple laptops). To downweight them, the authors download up to five still images per object state via Google Image Search and compute a per-video relevance score rv as the max similarity between exemplar-image features and video-frame features (in a pretrained visual embedding), constrained by the same causal ordering. A sigmoid of rv gives a scalar weight ω(v) ∈ [0,1] that scales each video's loss contribution — near 0 for irrelevant videos, near 1 for relevant ones. This handful of exemplar images is the only manual data in the whole pipeline; notably they use exemplar state images, not action images.

3. Architecture. Both g and h are two-layer MLPs (hidden 512, ReLU) over concatenated frozen 2D (ResNeXt, ImageNet-21K) + 3D (TSM ResNet-50, HowTo100M + AudioSet) features at one frame per second; feature extractors are not backpropagated through. Unlike Alayrac et al.'s linear/closed-form classifiers, the non-linear MLP and SGD training are what let it scale to noisy large data.

Setup

  • Datasets / benchmarks: New ChangeIt dataset — 44 state-changing actions, 34,428 untrimmed in-the-wild YouTube videos, 2,642 total hours, 4.6 min average duration; 667 videos (48 hrs, ~15 per class) manually annotated with background / initial state / action / end state labels for evaluation. Also evaluated on the smaller curated dataset of Alayrac et al. [2] (7 trimmed object-action classes).
  • Hardware / simulator: not reported (purely passive video understanding; no robot, no simulator).
  • Baselines: Alayrac et al. [2] (linear discriminative clustering, prior SOTA); zero-shot CLIP ViT-L/14 [52] with prompt engineering; MIL-NCE S3D [42] (HowTo100M); image-exemplar-only similarity; Random / Random-with-constraint chance baselines.
  • Compute: not reported (100 training epochs; feature extractors frozen).

Results

Metric: per-video action precision (1/0 whether predicted action location falls in the GT interval) and state precision (0 / 0.5 / 1 for how many of the two state locations match), averaged within then across categories, following [2].

  • On Alayrac et al.'s own dataset (Table 1): Ours St 0.49 / Ac 0.58 vs. Alayrac et al. St 0.48 / Ac 0.55 and CLIP ViT-L/14 St 0.42 / Ac 0.42 — beating the prior SOTA despite [2] using stronger supervision (a pretrained object detector with known bounding boxes).
  • On the new ChangeIt dataset (Table 2): Ours St 0.35 / Ac 0.68, clearly ahead of Alayrac et al. (0.30 / 0.59), CLIP ViT-L/14 (0.30 / 0.63), MIL-NCE S3D (0.27 / 0.50), image-examples (0.29 / –), and Random-with-constraint (0.15 / 0.41) — demonstrating the method scales to noisy untrimmed video where the baselines do not.
  • Ablations (Fig 3, Table 2c): performance rises monotonically with dataset size (no saturation at 34k videos — larger would help). Removing noise-adaptive weighting drops Ac 0.68→0.64; removing the multi-layer MLP (linear, as in [2]) drops to 0.61; removing data augmentation 0.66. Hard-negative sampling at κ ≥ 60s strongly beats random negatives (Fig 3b).

Limitations & open questions

From the authors:

  • Relies on training videos existing on the web — common or unintentional daily actions (e.g. "open a fridge") may be rare on YouTube.
  • Some object-state appearance variation is too large to be learned unsupervised (their example: "remove weed").
  • A separate model is trained per action class; sharing information across related classes (e.g. "cut apple" and "cut avocado") is an open problem.

What I noticed reading it:

  • The model localizes a single most-prominent initial→end transition per video; multi-step or repeated state changes within one video (a real recipe) are explicitly treated as background — it is not a continuous process monitor.
  • The "state" is only ever a discrete pair (initial vs. end) tied to one named action; there is no notion of a continuous or graded state ("how cooked", "how full"), nor of state transitions driven by anything but the one labeled action.
  • The causal-ordering bias is purely visual and temporal — it never observes why the change happened (heat, force, wetting); state and cause are entangled in appearance only.
  • Per-class models plus frozen features mean nothing learned here closes any loop or produces a control signal; it is a labeller of past video, not a monitor of an ongoing process.

Why I care

This is the canonical object-state-change-from-video anchor — and precisely the conceptual baseline the dynamic-state-change idea departs from. Look for the Change owns the state-change-from-video axis: it is the paper that introduced ChangeIt and showed that initial state → action → end state can be learned at web scale from the arrow of time. But against the idea of grounding dynamic, change-oriented manipulation concepts on multimodal non-visual sensing and using them to drive closed-loop correction, it sits on the far side of every axis:

  • Static-pair vs. dynamic-transition: it does capture a change (initial vs. end), so it is closer to dynamic than static-property work like Octopi — but only as a discrete before/after pair, never as a continuous process ("coming to a boil", "became slippery after wetting").
  • Vision-only vs. non-visual sensing: entirely RGB(+motion) web video. The states it learns are exactly the ones that are visually apparent; the predicates the idea cares about ("hot enough now", "screwed tight") often live in touch / audio / thermal / force, which this paper never touches.
  • Passive recognition vs. closed-loop correction: it labels already-recorded video; it has no actuator, no policy, no replanning. Contrast with BLADE, whose whole point is precondition/effect predicates feeding a planner that replans after each behavior.
  • Self-supervised neural scores vs. symbolic predicates: the "state" here is a continuous classifier output with no symbolic identity or planning role — the opposite of BLADE's learned turned-on(faucet) predicate that gates an operator.

So its value to the thesis is as the scale-and-self-supervision reference: the proof that state changes can be mined from cheap web video with near-zero labels. The research bet is that the interesting manipulation state changes are the ones this paper structurally cannot see (non-visual) and cannot use (passive). It pairs naturally with the cooking-state line (Continuous Object State Recognition, Heat-Induced Food State Changes) and the event-grounding line (DCL) as the vision-only end of the spectrum.

Related ingested papers (this batch + corpus):

  • BLADE — the anchor: learns symbolic precondition/effect predicates for closed-loop planning; Look for the Change is the passive, vision-only, self-supervised contrast on what a learned "object state" can be.
  • Dynamic Concept Learner (DCL) — the other vision-only state/event-grounding paper in this batch; grounds physical concepts via dynamic visual reasoning rather than arrow-of-time self-supervision.
  • Continuous Object State Recognition (cooking, VLM) and Heat-Induced Food State Changes (time-series VLM) — push exactly the continuous, monitored-over-time state axis that Look for the Change's discrete initial/end pair stops short of.
  • Octopi — the non-visual (tactile) static-property contrast; together they bracket the vision/non-visual and static/dynamic corners around the idea.

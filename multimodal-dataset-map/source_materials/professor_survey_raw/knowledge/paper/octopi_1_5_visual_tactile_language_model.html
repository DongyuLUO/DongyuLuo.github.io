Demonstrating the Octopi-1.5 Visual-Tactile-Language Model

Samson Yu, Kelvin Lin, Harold Soh · National University of Singapore (Dept. of Computer Science / NUS Smart Systems Institute) · RSS 2025 (Demonstration track) · arXiv:2507.09985 · PDF · code

One-liner. Octopi-1.5 is a Qwen2-VL-7B visual-tactile-language model that turns GelSight tactile-video frames into tokens, reasons about object properties (hardness, roughness, texture) in language, and adds a simple retrieval-augmented generation (RAG) module so it can recognize unseen objects on-the-fly — demonstrated live through a handheld UMI-style "Tactile Manipulation Interface" gripper, no robot required.

Problem & motivation

Touch carries object properties (softness, deformability, surface texture, hardness) that vision can't read, and matters most exactly when vision is occluded (e.g., an object grasped inside a closed hand). Recent visual-tactile-language models (VTLMs) like the authors' own Octopi, TVL, and UniTouch fuse tactile, visual and language modalities, but they (a) are trained on limited data, (b) treat objects as single-part wholes, and (c) cannot incorporate new objects without retraining. This demonstration paper presents an improved model that processes tactile signals from multiple object parts and uses RAG to both boost performance and learn new tactile-object pairings at inference time.

Method

Octopi-1.5 has three pieces: a fine-tuned VLM backbone, a tactile encoder, and a retrieval module (architecture in Fig 2A).

Backbone. The model is built on Qwen2-VL 7B, an open-source VLM (the previous Octopi used Llama). Word, vision, and tactile inputs are each turned into tokens and processed by the decoder transformer.

Tactile encoder. Tactile frames from a GelSight Mini sensor are translated into tokens by a fine-tuned CLIP (L/14) module augmented with a projection layer. To save compute, only the top-10 salient frames are processed — selected by a heuristic that picks the frames with the largest differences from their preceding frame (i.e., the moments of greatest tactile change). The CLIP module is pre-trained with two heads (Fig 2B): a contrastive loss (distinguishing tactile inputs of the same object/part — positive class — from other objects/parts; positive and negative classes sampled from the same GelSight pad type, marker or markerless, for consistency) and a regression loss predicting human-annotated hardness and roughness scores. Annotators score hardness and roughness on a 0/5/10 scale with anchor examples (Table II: cotton ball / foam mattress / rock for hardness; ice / jeans / toothbrush bristles for roughness).

Two-stage training. First the CLIP module inside the tactile encoder is trained (30 epochs, best encoder chosen on validation); then the CLIP module is frozen and the projection layers are trained alongside the decoder via end-to-end fine-tuning. Decoder training uses Visual Prompt Tuning (VPT) on describe-and-rank tasks (Fig 3): the model is asked to describe and rank multiple parts of an object (and multiple objects) in decreasing hardness/roughness — this multi-part supervision is what lets Octopi-1.5 handle objects with distinct parts (e.g., a hairbrush handle vs. its bristles), unlike its predecessor.

RAG module. A deliberately simple text-retrieval scheme. For a new set of salient tactile images, compute their average embedding via the tactile encoder, do a cosine-similarity search over an existing dataset of tactile embeddings to find the top-5 most similar objects, aggregate unique objects ranked by number of retrieved samples, and append their labels + tactile descriptions to the prompt as a prioritized list of matches (example in Fig 4, augmentation shown in blue). Crucially this database can be extended on-the-fly ("teaching"): new object samples are added to the database without retraining the VLTM, enabling new-object recognition at inference time.

Tactile Manipulation Interface (TMI). A modified UMI-style (Universal Manipulation Interface) handheld gripper (Fig 5) with one finger carrying a GelSight Mini (high-res optical tactile imaging) and the other a TAC-02 piezoresistive sensor (64 taxels, lower resolution but 1 kHz sampling, captures static and dynamic pressure directly). The demo runs Octopi-1.5 on a remote workstation; users interact through a chat interface, grasping objects with the TMI to feed tactile data (no visual input during grasping).

Setup

  • Datasets / benchmarks: Training data (Table I) combines PhysiCLeAR-Plain (100 objects, 2689 tactile videos), PhysiCLeAR-Dotted (68 objects, 1939 videos), Hardness (210 objects, 1860 videos), and ObjectFolder-Real (100 objects, 3550 videos) — an expanded PhysiCLeAR including both marker and markerless GelSight pads, plus hardness [15] and ObjectFolder [13] datasets. Demo evaluation uses three object categories: balls (baseball, plush ball, tennis ball), fruits (apple, partially-ripe kiwi, orange), and completely unseen objects (hairbrush bristles, hairbrush handle, microfiber cloth).
  • Hardware / simulator: GelSight Mini (optical) + TAC-02 (piezoresistive, 64 taxels, 1 kHz) tactile sensors mounted on a UMI-derived TMI handheld gripper; GoPro camera for vision; laptop + remote workstation/server at NUS. No robot — this is a human-held tactile interface. Portable, assembled in under 15 minutes.
  • Baselines: Encoder-1.5 (non-VLM baseline using cosine similarity of tactile embeddings only); Octopi-1 (7B); +-Octopi-1 (13B); Octopi-1.5 (8B) without RAG; Octopi-1.5 (8B) with RAG (and with teaching).
  • Compute: not reported (CLIP module trained 30 epochs; full training-compute figures not given).

  • Octopi-1.5 (no RAG) already beats both Octopi-1 variants on balls (56 vs 44/48) and fruits (57.69 vs 42.31/34.62), though it slightly trails the 13B Octopi-1 on unseen objects (41.46 vs 53.66) — a place it loses without RAG.
  • Adding RAG lifts seen-object accuracy to 96–100% and unseen to 73.17%; enabling teaching (adding samples to the DB) pushes unseen to 95.12%.
  • The non-VLM Encoder-1.5 baseline is strong on familiar objects (80% balls, 100% fruits) but offers no out-of-the-box handling of unseen items without database updates — the VLM's value is commonsense reasoning + on-the-fly learning, not raw embedding matching.
  • Sorting task: Octopi-1.5 sorts balls (100%) and fruits (93.18%) by hardness well but struggles on unseen objects (43.33%); sorting by roughness "remains error-prone."

Limitations & open questions

From the authors:

  • Work-in-progress; general improvements needed on several tasks (e.g., sorting by roughness remains error-prone). Larger base LLMs might help but cost compute.
  • RAG is based solely on text retrieval; retrieving tactile embeddings directly may help but it's unclear what information is best to retrieve cheaply.
  • Octopi-1.5 only outputs text tokens — no direct link to manipulation actions. A Vision-Tactile-Language-Action (VTLA) model is under development.
  • Whisper speech-to-text was unreliable with soft-speaking participants in noisy environments, so may be dropped from the live demo.
  • Practical TMI issues: sensor wear-and-tear, annotation difficulty, variability across different tactile sensors.

What I noticed reading it:

  • This is a demonstration paper, not a controlled benchmark study: Table III numbers come from "preliminary tests" over modest sample counts (25/26/41) on a handful of objects per category — no seeds, no variance, no held-out test protocol. The gains are suggestive, not statistically established.
  • The headline 95–100% accuracies hinge on RAG over a curated database; with teaching, the model has effectively seen the object's tactile signature, so "unseen (teaching)" is closer to nearest-neighbor retrieval than zero-shot generalization. The honest zero-shot number is the 41.46% (no RAG) / 73.17% (RAG, no teaching) column.
  • The salient-frame heuristic (top-10 by frame difference) is unvalidated — no ablation on how many frames, or whether max-difference frames are the most informative tactile moments.
  • The TAC-02 piezoresistive sensor is integrated in hardware but the model "primarily" uses GelSight (more training data); multi-sensor fusion is aspirational here, not demonstrated quantitatively.
  • Hardness/roughness ground truth is human-annotated on a coarse 0/5/10 scale — a noisy regression target that bounds how fine-grained the learned property estimates can be.

Why I care

This sits squarely on the thesis that many manipulation predicates are not visually evaluable — surface_is_rough, is_hard, is_ripe, is_deformable live in touch, not pixels. Octopi-1.5 is a concrete instance of grounding such property predicates in tactile signals and exposing them through language — exactly the kind of perceptual classifier BLADE learns for its predicates, but for properties BLADE's visual open-vocabulary classifiers fundamentally cannot read. Two threads connect:

  • Predicate grounding via touch. BLADE learns fθ(p): O → {T,F} over RGB-D observations; an Octopi-style tactile encoder could supply the same interface for touch-dependent predicates, extending BLADE's abstraction layer to contact-rich/occluded states. The multi-part description capability is also relevant — predicates often attach to object parts (handle vs. bristles).
  • RAG-as-predicate-extension. The on-the-fly "teaching" idea (add a tactile exemplar, recognize a new object without retraining) is a lightweight answer to BLADE's flagged predicate-invention gap: rather than inventing a new symbol, attach a new grounded exemplar. Worth thinking about whether retrieval-augmented grounding could let a planning system acquire new perceptual predicates incrementally.

Caveat on relevance: Octopi-1.5 is a perception/description model with no action layer (the authors note VTLA is future work), so it is upstream of BLADE's planning concerns, not a planning paper. The connection is the predicate-grounding substrate, not composition.
